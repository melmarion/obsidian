## PHASE 1: IDEA VALIDATION (REDDIT METHOD)

### 1.1 THE REDDIT VALIDATION FRAMEWORK

**Step 1: Find Your Subreddit**

```
Target: Niche subreddit (not massive, 10K-100K members ideal)
Example: r/baseballcards (smaller, focused community)

Why Small Subreddits Work Better:
- Less competition for attention
- More engaged members
- Easier to avoid "self-promotion" bans
- Members genuinely want solutions
```

**Step 2: Make Validation Post (BEFORE Building)**

```
Post Template:

Title: "[Problem] - Would you use an app for this?"

Body:
"Hey r/[subreddit],

I've been [experiencing this problem] and couldn't find 
a good solution. I'm thinking about building an app that 
[solves specific problem].

Would this be useful to you? What features would you want?"

Rules:
- DON'T mention you're building it yet
- Frame it as a question/problem
- Be genuinely part of the community (or become one)
- Ask specific questions about features
```

**Validation Metrics:**

```
Success = 10-15+ engaged replies saying "I'd use this"
Kyle's result: 12-15 replies on validation post
This gave "decent amount of validation" to proceed
```

---

### 1.2 THE LAUNCH POST FORMULA

**Timing:** After you've built the MVP

**Post Structure:**

```
Title: "I built the [X] app I posted about a few months ago"

Body:
"Hey r/[subreddit],

A few months ago I made a post asking if people would be 
interested in [solution]. I got great feedback, so I went 
ahead and built it!

[App Name] is now live: [Link]

Features:
- [Feature 1]
- [Feature 2]  
- [Feature 3]

It's $X.XX on the App Store. Would love to hear your thoughts!"

Critical Elements:
✅ Reference your validation post (builds continuity)
✅ Thank the community for feedback
✅ Link directly to App Store
✅ Don't be salesy - just factual
✅ Respond to every comment
```

**Kyle's Results:**

- Launch day: 12 paid downloads
- Price: $4.99 (paid download, no subscription)
- Revenue: $60 day one
- Source: Single Reddit post, r/baseballcards

**Why This Worked:**

1. Pre-validated with community
2. Didn't feel like an ad (genuine member)
3. Solving real problem they expressed interest in
4. Already had social proof from validation post

---

### 1.3 ONGOING REDDIT STRATEGY

**Post Update Pattern:**

```
Every major update, make a new post:

"Update: [App Name] now has [new feature]"

Body:
"Thanks to everyone who's been using [app]. Based on your 
feedback, we just added:
- [Feature 1]
- [Feature 2]

Still available at [link]. Let me know what you think!"

Frequency: Every 2-4 weeks with significant updates
```

**Avoiding Bans:**

- Post genuinely useful updates (not just "buy my app")
- Engage in subreddit beyond your posts
- Some subreddits are stricter (r/footballcards banned them, r/baseballcards didn't)
- If you get banned from one, pivot to another related subreddit
- "I knew I was building something that directly helped people" = justification

**Reality Check:**

```
"None [of the Reddit posts] got that many views, but knowing 
what I know now about hook optimization, I didn't know anything. 
I was just like 'let me make an engaging title and hope it works.'"

Translation: Don't expect massive numbers from Reddit.
It's about getting your first 10-50 users, not 10,000.
```

### 2.2 KEYWORD RESEARCH PROCESS (FREE METHOD)

**Step 1: Brainstorm Keyword List**

```
Think like your user searches:

For baseball card app:
- "baseball card scanner"
- "find value for baseball cards"  
- "baseball card value"
- "sports card scanner"
- "card price checker"
- etc.

Rules:
- Use exact phrases users would type
- Think problem-first (not brand-first)
- Include variations (baseball/sports, scanner/finder)
- Aim for 20-30 initial ideas
```

**Step 2: Use Sensor Tower (Free Trial)**

```
Tool: Sensor Tower (sensortower.com)
Free trial: Use multiple emails for multiple trials

Process:
1. Input each keyword
2. Check:
   - Search volume (popularity)
   - Competition level (difficulty)
   - Current top apps for that keyword

3. Sort by:
   - Highest volume
   - Lowest competition
   - Most relevant to your app

Pick top 5-10 keywords
```

**Step 3: Alternative Free Tools**

```
- App Figures (appfigures.com)
- Astro (Mac app for tracking)
- Apple Search Ads Keyword Tool (free with account)

Kyle's method: 
"Did free trial of Sensor Tower, looked at all top keywords 
related, brainstormed list, checked rankings, picked highest 
value ones"
```

---

### 2.3 KEYWORD IMPLEMENTATION

**Where to Place Keywords:**

**App Title (Most Important):**

```
BEFORE ASO:
"Cardstock"

AFTER ASO:  
"Cardstock - Sports Card Scanner"

Rules:
- Primary keyword in title ALWAYS
- Keep under 30 characters if possible
- Make it readable (not keyword stuffed)
- Example: "Cardstock sports card scanner"
```

**Subtitle (Second Most Important):**

```
Kyle's subtitle: "Find value for baseball cards"

Rules:
- Secondary keyword phrase
- Should complete the title's promise
- Under 30 characters
- Natural language, not stuffed
```

**Keyword Field (100 character limit):**

```
Use comma-separated keywords (no spaces after commas):
"baseball,card,value,scanner,price,checker,pokemon,collection"

Rules:
- Don't repeat words from title/subtitle
- No spaces (wastes characters)
- Prioritize by search volume
- Test variations over time
```

**The Priority Rule:**

```
"Order keywords by their priority and what you want to do."

Ranking:
1. Title > 2. Subtitle > 3. Keyword field

Put highest volume keyword in title, second highest in subtitle.
```

### 4.3 THE WINNING FORMAT (VALUE BIN/CARD SHOW)

**Format Discovery:**

```
Went to sports card shows (conventions where people buy/sell cards)
Realized: Cardstock actually helps make better deals in real-time
Found: Videos from card shows perform well on TikTok

Combined: Show real use case + demonstrate app value
```

**The Winning Formula:**

**Structure (3 Key Elements):**

```
1. HOOK (3-5 seconds):
   - Show desirable card close-up
   - On-screen text: "What is this [Player Name] card worth?"
   - Creates curiosity/suspense

2. DEMONSTRATION (3-5 seconds):
   - Phone in hand (visible on screen)
   - Scan the card with app
   - Show the app interface clearly

3. REVEAL (2-3 seconds):
   - Card value appears on screen
   - Often surprising (high value or low value)

Total video length: 8-15 seconds
```

**Example Script/Voiceover:**

```
Jacob's voiceover style:

"I was at a card show in Orlando and I found this Michael 
Jordan card. [scans with app] It's worth $500, but they're 
selling it for $30. I bought it for $30 and flipped it next 
show for $500."

Elements:
- Storytelling (location, discovery)
- Suspense (what's it worth?)
- Payoff (value reveal)
- Proof of concept (I actually bought/sold it)
```

**Visual Requirements:**

```
✅ Phone physically in frame (product placement)
✅ App logo visible in scanner (added "Cardstock" branding to scanner UI)
✅ Smooth scanning motion
✅ Clear value reveal
✅ Real cards (not stock photos)
```

```
- Iterate on what works
```

## PHASE 6: SCALING WITH NOISE PLATFORM

### 6.1 WHAT IS NOISE?

**Platform Overview:**

```
Noise = Creator distribution platform for slideshows/UGC

How it works:
1. Brands upload content template
2. Noise auto-distributes to creator network
3. Creators post on their personal TikTok accounts
4. Brand pays per 1,000 views (CPM model)

Noise's business model:
- Charges brand $2 CPM
- Pays creators $1 CPM
- Keeps $1 CPM (50% cut)
```

**Why Noise Exists:**

```
Solves major scaling problems:
- Finding creators (Noise has network)
- Negotiating rates (standardized CPM)
- Content quality (templates provided)
- Payment/tracking (automated by platform)
- Content distribution (happens automatically)

Alternative: Hire creators individually
Problems: Outreach, onboarding, briefing, feedback loops, payment
Time sink: Massive
```

---

```
In Noise dashboard:
```

**Bulk vs Viral Strategy:**

```
Kyle: "Definitely one outperforming video is way better than bulk 
of small videos."

But: "Bulk of small videos is still good. They still convert."

Why bulk works:
- Total reach adds up (7,600 posts × average 3,000 views = 23M views)
- More chances for algorithm to pick one up
- Consistent presence (users see multiple posts)
- Diversified risk (not relying on one hit)

Why single viral video is better:
- Social proof (high engagement = trust)
- Algorithm boost (TikTok pushes it further)
- Comments drive more organic reach
- "If you see a video with this many likes, bookmarks, shares, 
  comments - you trust the knowledge is good"
```

### PROMPT 2: ASO KEYWORD RESEARCH

**Step-by-Step Process:**

```
1. Brainstorm 20-30 keyword phrases users would search
   - Think problem-first (not brand-first)
   - Include variations and synonyms

2. Sign up for Sensor Tower free trial (or use multiple emails)

3. For each keyword, check:
   - Search volume (popularity score)
   - Difficulty (competition level)
   - Currently ranking apps

4. Sort by: High volume + Low competition + Relevant

5. Pick top 5-10 keywords

6. Implementation:
   - #1 keyword → Title
   - #2 keyword → Subtitle
   - #3-10 → Keyword field (comma-separated, no spaces)

7. Track rankings weekly (use Astro or Sensor Tower)

8. Iterate: If not ranking after 2 weeks, adjust keywords
```
